第2章 搜索业务规则分析

2.1需求分析

我们今天要完成的目标是在关键字搜索的基础上添加面板搜索功能。

面板上有商品分类、品牌、各种规格和价格区间等条件

业务规则：

  • 当用户输入关键字搜索后，除了显示列表结果外，还应该显示通过这个关键字搜索到的记录都有哪些商品分类。
  • 根据第一个商品分类查询对应的模板，根据模板查询出品牌列表。
  • 根据第一个商品分类查询对应的模板，根据模板查询出规格列表。
  • 当用户点击搜索面板的商品分类时，显示按照这个关键字查询结果的基础上，筛选此分类的结果。
  • 当用户点击搜索面板的品牌时，显示在以上结果的基础上，筛选此品牌的结果。
  • 当用户点击搜索面板的规格时，显示在以上结果的基础上，筛选此规格的结果。
  • 当用户点击价格区间时，显示在以上结果的基础上，按价格进行筛选的结果。
  • 当用户点击搜索面板的相应条件时，隐藏已点击的条件。

2.2 实现思路

  • 搜索面板的商品分类需要使用Spring Data Solr的分组查询来实现
  • 为了能够提高查询速度，我们需要把查询面板的品牌、规格数据提前放入redis
  • 查询条件的构建、面板的隐藏需要使用angularJS来实现
  • 后端的分类、品牌、规格、价格区间查询需要使用过滤查询来实现

修改SearchItemServiceImpl.java创建方法

/****
 * 查询所有分类数据
 * @param query
 * @return
 */
public List<String> searchCategory(Query query){
    //设置分组信息
    GroupOptions groupOptions = new GroupOptions();

    //添加分组域
    groupOptions.addGroupByField("item_category");

    //设置分组选项
    query.setGroupOptions(groupOptions);

    //实现分组查询
    GroupPage<Item> groupPage = solrTemplate.queryForGroupPage(query, Item.class);

    //获得指定域的分组数据
    GroupResult<Item> groupResult = groupPage.getGroupResult("item_category");

    //获得分组数据的入口，通过它可以获得分组数据
    Page<GroupEntry<Item>> groupEntries = groupResult.getGroupEntries();

    //循环获得分组数据
    List<String> list = new ArrayList<String>();
    for (GroupEntry<Item> groupEntry : groupEntries) {
        list.add(groupEntry.getGroupValue());
    }
    return list;
}

第3章 缓存品牌和规格数据

3.1需求分析

将商品分类数据、品牌数据、和规格数据都放入Redis存储。

（1）当用户进入运营商后台的商品分类页面时，将商品分类数据放入缓存（Hash）。以分类名称作为key ,以模板ID作为值

（2）当用户进入运营商后台的模板管理页面时，分别将品牌数据和规格数据放入缓存（Hash）。以模板ID作为key,以品牌列表和规格列表作为值。

3.2缓存商品分类数据

将商品分类表存入缓存 pinyougou-sellergoods-service工程需要引入pinyougou-common工程依赖。同时注意导入相关配置文件。

在spring.xml中引入redis配置

<import resource="spring-redis.xml" />

修改pinyougou-sellergoods-service的ItemCatServiceImpl.java，注入redisTemplate，把分类信息存储到Redis缓存中。key为分类的名字，value为分类对应的模板ID。

@Autowired
private RedisTemplate redisTemplate;

/***
 * 根据父节点ID查询子类分类
 * @param id
 * @return
 */
@Override
public List<ItemCat> findByParentId(Long id) {
    //select * from tb_item_cat where parent_id=parentId;
    ItemCat itemCat = new ItemCat();
    itemCat.setParentId(id);

    //重新存入缓存
    List<ItemCat> itemCats = itemCatMapper.selectAll();
    for (ItemCat cat : itemCats) {
        //缓存的key为分类名字，value为分类的模板ID
        redisTemplate.boundHashOps("ItemCat").put(cat.getName(),cat.getTypeId());
    }
    return itemCatMapper.select(itemCat);
}

3.3缓存品牌和规格列表数据

修改pinyougou-sellergoods-service的TypeTemplateServiceImpl.java

3.3.1 刷新缓存方法编写

添加一个刷新缓存的方法，主要用于刷新品牌和规格。其中品牌信息直接取表中brand_ids列对应的数据，而规格可以直接使用之前写好的添加商品时查询规格的方法。

@Autowired
private RedisTemplate redisTemplate;

//刷新缓存
public void refreshRedis(){
    //获取所有模板
    List<TypeTemplate> typeTemplates = typeTemplateMapper.selectAll();

    //循环存储所有模板信息
    for (TypeTemplate typeTemplate : typeTemplates) {
        //将品牌信息转成List<Map>
        List<Map> brandList =JSON.parseArray( typeTemplate.getBrandIds() ,Map.class);

        //将typeTemplate的ID作为key,品牌信息作为value
        redisTemplate.boundHashOps("BrandList").put(typeTemplate.getId(),brandList);

        //将typeTemplate的ID作为key,规格信息作为value
        redisTemplate.boundHashOps("SpecList").put(typeTemplate.getId(), getSpecOptionsByTypeId(typeTemplate.getId()));
    }
}

说明：getSpecOptionsByTypeId(typeTemplate.getId())也就是前面我们拼接一个如下JSON格式规格数据的方法,前面你的方法在哪个类则灵活调用哪个类。之前调用生成规格数据格式如下：

[

   {"id":2,"text":"机身内存","options":[{"id":1233,"optionName":"4G","8G"}]},
   {"id":3,"text":"操作系统","options":[{"id":1234,"optionName":"Android","IOS"}]}
]
3.3.2 调用刷新缓存方法

在查询分页方法(getAll) 时调用此方法

/***
 * 分页返回TypeTemplate列表

@Override
public PageInfo<TypeTemplate> getAll(TypeTemplate typeTemplate,int pageNum, int pageSize) {

4.2.1 实现品牌+规格搜索

修改ItemSearchServiceImpl.java ，增加方法

该方法主要用于根据分类的名称从缓存中获取之前存储在缓存中的品牌信息和规格信息。

/***
 * 根据分类名称获取品牌和规格信息
 * @param category
 * @return
 */
public Map<String,Object> specAndBrand(String category){
    //存储品牌和规格信息
    Map<String,Object> dataMap = new HashMap<String,Object>();

    //获得模板ID
    Long typeId = (Long) redisTemplate.boundHashOps(ItemCat.class.getSimpleName()).get(category);

    if(typeId!=null){
        //根据模板ID从缓存中获取对应分类的品牌列表
        List brandList = (List) redisTemplate.boundHashOps("BrandList").get(typeId);

        //根据模板ID从缓存中获取对应分类的规格列表
        List specList = (List) redisTemplate.boundHashOps("SpecList").get(typeId);

        dataMap.put("brandList",brandList);
        dataMap.put("specList",specList);
    }
    return dataMap;
}
4.2.2 search方法调用品牌+规格搜索方法

            query.addFilterQuery(filterQuery);
        }
        //品牌过滤
        String brand = (String) search.get("brand");
        if(StringUtils.isNotBlank(brand)){
            //创建过滤条件
            Criteria criteria = new Criteria("item_brand").is(brand);
            FilterQuery filterQuery = new SimpleFilterQuery(criteria);
            query.addFilterQuery(filterQuery);
        }

        //规格过滤

                query.addFilterQuery(filterQuery);
            }
        }

    }


	//...略
    dataMap.put("categoryList",categoryList);
    dataMap.put("rows",items);
    dataMap.put("total",total);
    return dataMap;
}
